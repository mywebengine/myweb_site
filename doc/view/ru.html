view - адаптация верстки

Для организации адаптации верстки приложения можно использовать реактивный объект view, который содержит информацию о ширине, высоте и позиции прокрутки окна. Этот объект изменяется при изменении ширины, высоты, или позиции прокрутки окна.

Используя этот объект можно реагировать на изменение размеров окна или измение позиции прокрутки и адаптировать интерфейс.

Структура объекта

  • clientWidth

    Ширина окна в пикселях, для теущего окна равна: ${view.clientWidth}

  • clientHeight

    Высота окна в пикселях, для теущего окна равна: ${view.clientHeight}

  • scrollTop

    Вертикальная позиция прокрутки в пикселях, для теущего окна равна: ${view.scrollTop}

  • scrollLeft

    Горизонтальная позиция прокрутки в пикселях, для теущего окна равна: ${view.scrollLeft}

Объект view обновляется в "экономичном" режиме - с задержкой. По этой причине, нужно использовать CSS для включения и выключения видимости элементов, которые меняет свое место положения исходя из размера экрана.

Пример

В данном примере демострируется следующие ситуации:

  1. При достижении значения вертикальной прокрутки значения в 70px (высота шапки 50px + отступ 20px), левое меню фиксируем к левой части окна, так что бы оно всегда было на виду

  2. При достижении окна ширины в 400px, левое меню перезжает в меню "бутерброд", а контентная часть занимает всю шарину
