filling - статуст загрузки

Команда управления статусом загрузки элемента. В зависимости от значения выражения статус загрузки будет показан или скрыт. Если значение истинно, то статус будет показан, если нет - скрыт. Выражение выполняется как js-код, в контексте текущего элемента.

Синтаксис

<div filling[.type.time]="expression">...</div>

Параметры

  • type необязательный

    Тип загрузчика, в комплекте есть только один варант реализации типа, название его spinner - это очень простая крутилка. О том как добавить собственные типы, обсуждается ниже.

  • time необязательный

    Время в мс через которое нужно показать загрузчик, если выражение будет ложно.

Описание

За визуализацию статуса загрузки отвечает CSS. Когда статус загрузки показывается: на элементе появляется атрибут is_filling="", если указан type, например spinner, то добавляется еще одни атрибудт is_filling-spinner="". В CSS нужно описывать правила для тегов с данными атрибутами. Стандартные парвила описываются в файле myweb.css.

Для атрибута is_filling="" описано правило, которое удаляет все дочерние элементы из области видимости:

*[is_filling] > * { display: none !important; }

Для типа spinner атрибута is_filling-spinner="", объявление правила выглядит так:

*[is_filling-spinner] { ... }

Для пользовательского типа с названием skeleton1 атрибутом которого будет is_filling-skeleton1="", объявление правила выглядит так:

*[is_filling-skeleton1] { ... }

Если у вас SPA то может возникнуть ситуация, что клиент видит загруженную HTML разметку которая еще не прошла рендер. И для того что бы скрыть шаблон можно указать атрибут is_filling[-type]="" сразу, не дожидаясь того когда это сделает команда filling.

<div is_filling="" filling="..." ...

Для того что бы сделать отображение скилета интерфейса на момент загрузки данных можно изсопльзовать такой поход:

  1. Выражение: проверять рективные переменные с данными, которые ожидаем
  2. Тип: название типа в соответствии с частью интерфейса для которого делается статус
  3. Время: на ваше усмотрение, так как дочерние элементы шаблона будут скрыты за display: none, то можно предположить что для более приятного повления интерфейса стоило бы заложить хотя бы пол секунды до применения скилета
